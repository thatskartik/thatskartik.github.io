CSES Introductory problems

  • Find the missing Number: Let \(g_1\) be a group of numbers and let \(g_2\) be another group of numbers such that \(g_2 \in g_1\). Then use the following property

    \[ g_1 \oplus g_2 = g_1 - g_2 \]

  • Adjacent difference 1 Permutation: Use Parity. First output all the even numbers and then all the odd ones
  • 2 Knights Don’t attack: If 2 KNights don’t attack each other, there won’t be a \(3 \times 2\) box in which they both can be contained. Secondly due to symmetry one can notice that the number of ways to do this is equal for each of the 8 attacking configs. The number of boxes of time \(m \times n\) that can be placed are \[ \mathcal{N} = (k - m - 1) \times (k - n - 1) \] And accounting for duplicates we divide by 2. So the final answer is \[ {k^2 \choose 2} - 4 \times \mathcal{N} \]
  • Trailing Zeroes: Number of Trailing zeroes is the number of factors of 5 (since it is the bottleneck), to compute that just evaluate \[ \sum_{ i = 1}^{\infty} \left \lfloor\frac{n}{5^i} \right \rfloor \]
  • Finishing coins with moves Solve the set of equations given by \[ a = 2m_1 + m_2 \\ b = 2m_2 + m_1 \] and impose the condition that \(m_1, m_2\) are positive integers
  • Reorder Palindrome: Reorder a string to make it a palindrome. The solution is to simply maintain a freq table of the seen characters and then use two pointers to construct the string.

  • Gray Code: Each successive string must differ only by a single bit. To do this , think of a recursive solution. Suppose you have a gray code of length 2. Then how do you make one for 3? Use the following ideaThe overline means you reverse the set of strings (not the strings themselves) in the previous gray code.

    \[ gc(n) = \{0 + gc(n - 1), 1 + \overline{gc(n - 1)} \} \]

  • Tower of Hanoi : Let us call the three sticks source, inter, and target. Approaching it using recursion, if we have a strategy to transfer \(n -1\) disks from stick \(A\) to stick \(B\) using an intermediate stick, then we first transfer the \(n- 1\) smallest disks to inter using target. Then we push the biggest onto target and again move the top \(n -1\) to target using source as the new intermediate stick.
  • Split into 2 groups with Minimal distance: The solution does not seem obvious at all. One could try sorting and then using the reverse and add technique to make groups on both sides. But the number of elements in the groups may not be equal. Since the contraints are not strict, we can afford trying every subset possible and evaluating the minimum value. The one closest to \(S/2\) where \(S\) is the sum will clearly win. \[ \text{min} \ | x - (S - x) | = | 2x - S | \]

  • Permutations with Repititions: This problem is an interesting one. We ae to generate permutations of a string with repititions. This is related to the general Idea of avoiding repitions that is described in the Neetcode problems on Backtracking.In this problem, I use the insertion technique to do permutations where \[ P_i = \text{Insert} \ a_i \rightarrow P_{i + 1} \]
    The general idea in this permuting is to make the decision of “swapping” elements and then you proceed by permuting the remaining up until the end, the recursion is non-linear. The algorithm is like:-

     
  permute(a, l, r){
      //base case
      if(l == r){
          res.push_back(a);
          return;
      }

      for(i = l to r){
          swap(a_i, a_l)

          permute(a, l+1, r)

          //reset
          swap(a_l, a_i);
      }
  }

    To make distinct permutations, we simply first sortSorting is always useful when you want to remove duplicates because similar characters are grouped together. In our case, if we swap with the same character again or if our iterate is the same as the primary swapping index, then we have a duplicate.
